360Works FMPro JDBC Driver

Why did we write a custom JDBC Driver?

  • Support for auto-entered serial #'s in FileMaker Pro
  • Support for dates in PreparedStatements
  • Multi-threaded and streaming, which means it can retrieve an unlimited number of records without running out of memory
  • Better support for container fields
  • Much, much smaller (about 100K as opposed to several megabytes for driver that ships with FileMaker Server)
  • More reliable, in my experience
  • Includes source code
  • Works through XML publishing, so does not require FMSA.
  • Does not require any server plugins, permissions, or configuration other than standard XML publishing
  • Supports more data types
  • Better error reporting

There are some downsides of our driver to be aware of compared to the FileMaker-supplied driver:

  • Not as fast for small record sets, updates, and inserts
  • Cannot create tables and fields on the fly
  • Cannot combine AND / OR searches (this is a limitation of XML gateway)

The JDBC driver is available as part of WooF, download it and try it out.
